Firewalld

FirewallD es un controlador frontend para la tabla iptables que se usa para implementar reglas de tráfico de red persistentes.

RHEL

La configuración específica de las redes en un servidor dependerá del uso que le vayas a dar, estos son pasos basicos que por lo general se usan en todos los servidores.

Instalar y gestionar FirewallD 🔧

1 - Primero verificamos que no este instalado en el sistema:

sudo systemctl status firewalld

2 - En aso que no se encuentre procedemos a instalar:

sudo dnf install firewalld -y

3 - Activamos el servivio:

sudo systemctl enable firewalld

4 - Iniciamos el servivio:

sudo systemctl start firewalld

Comandos basicos 💻

5 - Activar inicio automatico del Servicio:

sudo systemctl enable firewalld

6 - Iniciar Servicio:

sudo systemctl start firewalld

7- Status de Servicio:

sudo systemctl status firewalld

8 - Reiniciar Servicio:

sudo systemctl restart firewalld

9 - Detener Servicio:

sudo systemctl stop firewalld

10 - Desactivar inicio automatico del Servicio:

sudo systemctl disable firewalld

Debian

Estas instrucciones estan garantizadas para Sistemas Operativos basados en Debian, Ubuntu.

Configuración básica

Abrir puertos o servicios comunes.

Abrir puertos o servicios comunes 🔧

1 - Ejemplo: Permitir HTTP (80) y HTTPS (443):

sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https

2 - Ejemplo: Abrir un puerto específico (ej. 8080):

sudo firewall-cmd --permanent --add-port=8080/tcp

3 - Recargar el firewall para aplicar cambios:

sudo firewall-cmd --reload

Comandos basicos 💻

4 - Listar todas las reglas:

sudo firewall-cmd --list-all

5 - Listar zonas activas:

sudo firewall-cmd --get-active-zones

6 - Listar servicios permitidos:

sudo firewall-cmd --get-active-zones

7 - Listar puertos abiertos:

sudo firewall-cmd --list-ports

8 - Eliminar un puerto o servicio:

sudo firewall-cmd --permanent --remove-port=8080/tcp
sudo firewall-cmd --permanent --remove-service=http

Configurar zona predeterminada (opcional) 💻

9 - Ver zona actual:

sudo firewall-cmd --get-default-zone

10 - Cambiar zona predeterminada (ej. a public):

sudo firewall-cmd --set-default-zone=public

Desintalar Firewalld

Eliminar por completo firewalld y tod sus configuraciones.

Para eliminar Firewalld del sistema 🔧

1 - Detener el servicio de firewalld:

sudo systemctl stop firewalld

2 - Deshabilitar firewalld y evitar que se inicie automáticamente al arrancar el sistema

sudo systemctl disable firewalld

3 - Desinstalar el paquete de firewalld:

sudo dnf remove firewalld

4 - Eliminar también los archivos de configuración de firewalld:

sudo rm -rf /etc/firewalld

Debian

Estas instrucciones estan garantizadas para Sistemas Operativos basados en Debian, Ubuntu.
