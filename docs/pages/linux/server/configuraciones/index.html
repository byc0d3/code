La configuración específica del servidor dependerá del uso que le vayas a dar, estos son pasos basicos que por lo general se usan en todos los servidores.

Repositorios y paquetes escenciales 🔧

1 - Actualizamos el S.O y sus paquetes:

sudo dnf update && dnf upgrade

2 - Agregamos el repositorio EPEL:

sudo dnf install epel-release

3 - Instalamos paquetes escenciales:

sudo dnf install dnf-utils nano tree curl wget btop traceroute iproute telnet nmap tcpdump iputils unzip tar git tar rsync util-linux-user nfs-utils bind-utils

Activar rpmfusion 🔧

4 - habilitar el repositorio CRB:

sudo dnf config-manager --set-enabled crb

5 - Importar EPEL y EPEL Next Repository:

sudo dnf install \ https://dl.fedoraproject.org/pub/epel/epel-release-latest-9.noarch.rpm \ https://dl.fedoraproject.org/pub/epel/epel-next-release-latest-9.noarch.rpm

6 - Instale RPM Fusion Free:

sudo dnf install --nogpgcheck https://mirrors.rpmfusion.org/free/el/rpmfusion-free-release-$(rpm -E %rhel).noarch.rpm -y

7 - Instale RPM Fusion No Free:

sudo dnf install --nogpgcheck https://mirrors.rpmfusion.org/nonfree/el/rpmfusion-nonfree-release-$(rpm -E %rhel).noarch.rpm -y

SELinux (Security-Enhanced Linux) es un módulo de seguridad para el kernel de Linux que proporciona una capa adicional de seguridad sobre el sistema tradicional de control de acceso discrecional (DAC).

Archivos de Configuración 🔧

1 - En la etapa de configuración inicial del servidor, es recomendable establecer el SELinux en modo Permissive (para aplicar los cambios, deberá reiniciar su servidor), editamos:

sudo nano /etc/sysconfig/selinux

2 - Modificamos la siguente linea:

SELINUX=permissive

3 - Reiniciamos el sistema:

sudo reboot

La zona horaria es una región de la Tierra que tiene la misma hora estándar para fines legales, comerciales y sociales. Es una forma de estandarizar la medición del tiempo en diferentes ubicaciones geográficas, a medida que la Tierra gira y la posición del sol cambia.

Configurar Zona Horaria 🔧

1 - Para cambiar la hora y la zona horaria ejecutamos el siguiente comando, modificando segun sea el Pais/Estado:

sudo timedatectl set-timezone America/Caracas

2 - Si desea ennumerar todas las zonas horarias disponibles ejecutamos lo siguiente:

sudo timedatectl list-timezones

3 - Para sincronizar el tiempo, y habilitarlo ejecutamos::

sudo systemctl start chronyd && systemctl enable chronyd && systemctl status chronyd

RHEL

Un banner al iniciar sesión por SSH en tu servidor es una excelente manera de mostrar información importante, como avisos legales, políticas de uso o simplemente un mensaje de bienvenida personalizado.

Configurar Banner 🔧

1 - Para crear un banner que sea visible al conectarnos por ssh, editamos el siguiente archivo:

sudo nano /etc/motd

2 - Agregamos cualquier dieño, es este caso dejo esta plantilla:


                        |----------------------------------------------------------------|
                        | """                     SERVER-APP01-PRO                    "" |
                        | -------------------------------------------------------------- |
                        | -------------------------------------------------------------- |
                        | """ Si no tiene autorizacion para ingresar a este servidor """ |
                        | """ salga de inmediato, el mismo esta siento monitoriado   """ |
                        | """ se aplicaran severas sanciones por uso no autorizado.  """ |
                        | -------------------------------------------------------------- |
                        | -------------------------------------------------------------- |
                        | """    SEDE -  Direccion - Infraestructura - Servidores    """ |
                        | -------------------------------------------------------------- |

3 - Guardamos y salimos, al conectarnos nuevamente a servidor se vera el banner en la consola
