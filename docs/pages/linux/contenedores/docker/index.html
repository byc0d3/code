Docker

Docker es una plataforma de software que le permite crear, probar e implementar aplicaciones rápidamente. Docker empaqueta software en unidades estandarizadas llamadas contenedores que incluyen todo lo necesario para que el software se ejecute, incluidas bibliotecas, herramientas de sistema, código y versión ejecutable.

  1. Linux
  2. Docker

RHEL

Instalación 🔧

1 - Para instalar la ultima version de docker, primero instalamos dependencias necesarias:

sudo dnf install -y dnf-utils device-mapper-persistent-data lvm2

2 - Agregar el repositorio oficial de Docker:

sudo dnf config-manager --add-repo=https://download.docker.com/linux/centos/docker-ce.repo

3 - Instalar Docker Engine (Community Edition):

sudo dnf install -y docker-ce docker-ce-cli containerd.io

4 - Habilitar e iniciar el servicio Docker:

sudo systemctl enable docker --now

5 - Agregar tu usuario al grupo docker (para evitar usar sudo):

sudo usermod -aG docker $USER
newgrp docker

RHEL

Instalar Docker Compose 🔧

1 - Descarga la última versión desde el repositorio oficial:

sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

2 - Dar permisos de ejecución:

sudo chmod +x /usr/local/bin/docker-compose

3 - Verificar la instalación:

sudo docker-compose --version

Comandos basicos 💻

4 - Iniciar los contenedores en segundo plano:

sudo docker compose up -d

5 - Ver los contenedores en ejecución:

sudo docker ps

6 - Detener todos los contenedores:

sudo docker compose stop

7 - Ingresar el bash del contenedor:

sudo docker exec -it nombreContenedor /bin/bash

RHEL

Eliminar Docker y Docker Compose 🔧

1 - Detener y eliminar todos los contenedores en ejecución (opcional):

sudo docker stop $(sudo docker ps -aq)
sudo docker system prune -a --volumes

2 - Desinstalar paquetes de Docker:

sudo dnf list installed | grep docker
sudo dnf remove -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

3 - Eliminar archivos residuales de Docker:

sudo rm -rf /var/lib/docker
sudo rm -rf /var/lib/containerd
sudo rm -rf /etc/docker

4 - Eliminar Docker Compose, Si lo instalaste manualmente (con curl):

sudo rm -f /usr/local/bin/docker-compose
sudo rm -f /usr/bin/docker-compose

5 - Eliminar Docker Compose, Si lo instalaste desde el repositorio:

sudo dnf remove -y docker-compose-plugin

6 - Eliminar el repositorio de Docker (opcional):

sudo rm /etc/yum.repos.d/docker-ce.repo

7 - Eliminar el grupo docker (si lo creaste):

sudo groupdel docker

8 - Verificar eliminación:

docker --version
docker-compose --version

9 - (Opcional) Limpiar dependencias no usadas:

sudo dnf autoremove -y
