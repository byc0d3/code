PostgreSQL

PostgreSQL, también llamado Postgres, es un sistema de gestión de bases de datos relacional orientado a objetos y de código abierto, publicado bajo la licencia PostgreSQL, similar a la BSD o la MIT.

  1. Linux
  2. PostgreSQL

RHEL

Instalación 🔧

1 - Agregamos los repositorios de Redhat:

dnf install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-9-x86_64/pgdg-redhat-repo-latest.noarch.rpm

2 - Desactivamos el modulo por defecto de PostgreSQL:

dnf -qy module disable postgresql

3 - Instalamos PostgreSQL:

dnf install -y postgresql17-server postgresql17

4 - Iniciamos la base de datos:

sudo /usr/pgsql-17/bin/postgresql-17-setup initdb

Comandos basicos 💻

5 - Activar inicio automatico del Servicio:

systemctl enable postgresql-17

6 - Iniciar Servicio:

sudo systemctl start postgresql-17

7- Status de Servicio:

sudo systemctl status postgresql-17

8 - Reiniciar Servicio:

sudo systemctl restart postgresql-17

9 - Detener Servicio:

sudo systemctl stop postgresql-17

10 - Desactivar inicio automatico del Servicio:

sudo systemctl disable postgresql-17

RHEL

Querys Basicos 🔧

1 - Para conectarnos a la base de datos localmente:

sudo -u postgres psql

2 - Para conectarnos la base de datos remotamente:

psql -h host -U postgres -d postgres

3 - Crear base de datos con utf8_general:

CREATE DATABASE database ENCODING 'UTF8' LC_COLLATE 'en_US.UTF-8' LC_CTYPE 'en_US.UTF-8' TEMPLATE template0;

4 - Creamos un usuario:

CREATE USER user WITH PASSWORD 'passwd';

5 - Le damos permisos a ese usuario sobre esa base de datos:

GRANT ALL PRIVILEGES ON DATABASE database TO user;

6 - Para que un usuario Administrador tenga permisos sobre todas las bases de datos:

ALTER USER user WITH SUPERUSER;

RHEL

Configuraciónes 🔧

1 - Para permitir conexiones a la base de datos editamos el archivo postgresql.conf:

sudo nano /var/lib/pgsql/17/data/postgresql.conf

2 - Buscamos la linea listen_addresses y la dejamos de este modo:

listen_addresses = '*'

3 - Guardamos los cambios:

4 - Ahora editamos el archivo pg_hba.conf y agregamos los clientes a permitir:

nano /var/lib/pgsql/17/data/postgresql.conf

5 - Preferiblemente agregar una linea por cada cliente:

host	all		all		10.31.67.130/24		md5
host	all		all		10.31.225.99/32		md5

6 - Para cambair la clave del usuario postgres:

\password postgres

7 - Salimos:

\q

Extras

Extras 🔧

1 - Para crear un usuario para backups:

CREATE USER backup WITH PASSWORD 'passwd';

2 - Conceder permisos de SELECT sobre todas las tablas de una base de datos:


						GRANT CONNECT ON DATABASE database TO backup;
						\c database;
						GRANT SELECT ON ALL TABLES IN SCHEMA public TO backup;

3 - Debes repetir el proceso anterior para cada base de datos.
